# Blog Writer / 博客作者

[English](#english) | [中文](#中文)

---

## English

Write human-like blog posts without AI-generated patterns.

### What It Does

Creates natural, authentic blog content that sounds like a real person wrote it - avoiding common AI writing clichés and patterns.

### Core Principles

- Be specific, not generic
- Write like you talk
- Show, don't tell
- Vary sentence structure
- Use strong verbs
- Be opinionated

### Banned Words (AI Patterns to Avoid)

leverage, utilize, harness, empower, seamless, effortless, unlock, unleash,
dive into, landscape, ecosystem, paradigm, transformative, game-changing,
comprehensive, holistic, robust, synergy, alignment, pivotal, optimize,
enhance, boost, innovative, cutting-edge, state-of-the-art

### How to Use

```
/blog "How I built a side project in a weekend"
```

Or provide more context:

```
Write a blog post about learning Rust as a JavaScript developer.
Include my personal struggles and breakthroughs. Keep it casual and authentic.
```

### Output Structure

- Strong opening hook
- Personal experience
- Specific examples and data
- Actionable takeaways
- SEO-optimized format

### Example

```
You: /blog "Why I switched from VS Code to Neovim"

Claude: [Generates a personal, first-person blog post
        with specific details and natural voice]
```

## 中文

撰写自然流畅、避免 AI 写作痕迹的博客文章。

### 功能说明

创建自然、真实的博客内容，听起来像真人写的 - 避免常见的 AI 写作陈词滥调和模式。

### 核心原则

- 具体而非泛泛而谈
- 像说话一样写作
- 展示而非讲述
- 变换句式结构
- 使用强有力的动词
- 表达观点

### 禁用词汇（AI 模式词）

利用、运用、释放、解锁、无缝、简化、生态系统、范式、变革、革命性、
全面、整体、强大、协同、对齐、关键、优化、增强、提升、创新、前沿、最先进

### 使用方法

```
/blog "我是如何在一个周末内构建副业项目的"
```

或提供更多上下文：

```
写一篇关于 JavaScript 开发者学习 Rust 的博客文章。
包括我个人的挣扎和突破点。保持随意和真实的风格。
```

### 输出结构

- 强有力的开头钩子
- 个人经历
- 具体示例和数据
- 可行的要点
- SEO 优化格式

### 示例

```
你: /blog "为什么我从 VS Code 切换到 Neovim"

Claude: [生成一篇个人化、第一人称的博客文章，
        包含具体细节和自然语调]
```
